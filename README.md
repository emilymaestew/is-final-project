## ğŸš€ How to Upload to GitHub Pages

### Step 1: Create a GitHub Repository

1. Go to [GitHub.com](https://github.com) and log in with your account: **emilymaestew**
2. Click the **"+"** icon in the top right corner
3. Select **"New repository"**
4. Name your repository (e.g., `emilymaestew.github.io` or `is-final-project`)
5. Make sure it's set to **Public**
6. Click **"Create repository"**

### Step 2: Upload Your Files

**Option A: Using GitHub Web Interface (Easiest)**

1. On your new repository page, click **"uploading an existing file"**
2. Drag and drop ALL files and folders from the `emily-website` folder
3. Make sure to include:
   - `index.html`
   - `scratch.html`
   - `mealplanner.html`
   - `css/` folder
   - `js/` folder
   - `images/` folder
   - `assets/` folder
4. Write a commit message like "Initial website upload"
5. Click **"Commit changes"**

**Option B: Using GitHub Desktop (Recommended)**

1. Download and install [GitHub Desktop](https://desktop.github.com/)
2. Clone your repository to your computer
3. Copy all files from `emily-website` into the cloned repository folder
4. In GitHub Desktop, you'll see all the new files
5. Write a commit message and click **"Commit to main"**
6. Click **"Push origin"** to upload to GitHub

### Step 3: Enable GitHub Pages

1. Go to your repository on GitHub.com
2. Click **"Settings"** (top menu)
3. Scroll down and click **"Pages"** in the left sidebar
4. Under **"Source"**, select **"main"** branch
5. Click **"Save"**
6. Wait 1-2 minutes for the site to deploy
7. Your site will be live at: `https://emilymaestew.github.io/repository-name/`

## ğŸ†˜ Troubleshooting

### Images Not Showing
- Make sure all image files are in the `images/` folder
- Check that file names match exactly (case-sensitive)
- Verify the entire `images/` folder was uploaded to GitHub

### Tableau Not Loading
- Make sure you have internet connection (Tableau loads from external source)
- Check browser console for errors (F12 key)
- The Tableau script may take a few seconds to load

### GitHub Pages Not Working
- Wait 2-3 minutes after enabling Pages
- Make sure repository is set to Public
- Check that `index.html` is in the root directory
- Try accessing with `/index.html` at the end of URL

### Links Not Working
- Use relative paths (e.g., `scratch.html` not `/scratch.html`)
- Make sure all HTML files are in the root directory
- Test all links after deploying to GitHub Pages
